OpenAI's benchmark numbers for GPT-5 are impressive. Real-world performance is more complicated. I ran it through 14 tasks across coding, reasoning, and security analysis over three days. Here's what I found.

## Methodology

Tasks were run without system prompts to test baseline capability. Each task was attempted 3 times and scored on correctness, completeness, and whether the output required manual correction before use.

Models compared: GPT-5, Claude Sonnet 4.6, Gemini 2.0 Pro.

## Where GPT-5 wins

### Long-document reasoning

GPT-5's extended context handling is genuinely better than competitors on documents over 100k tokens. In a test where I fed it a 200-page technical specification and asked it to identify internal contradictions, it found 7 of the 9 I had planted. Claude found 6; Gemini found 5.

### Code generation for popular frameworks

For React, Next.js, and Python/FastAPI tasks, GPT-5 produced working code on the first attempt 11/14 times. It's clearly been trained on more recent library versions. It correctly used the Next.js App Router patterns without prompting, whereas Claude occasionally defaulted to Pages Router syntax.

### Mathematical reasoning

GPT-5 outperforms on multi-step math. Not relevant for most software tasks but notable for data-heavy applications.

## Where it falls short

### Security analysis

This was the most significant gap. When asked to identify vulnerabilities in a code sample containing a SQL injection, an SSRF, and a path traversal bug, GPT-5 found 2/3. It missed the SSRF entirely.

Claude found all three. For security-critical code review, this matters.

### Instruction following on constrained tasks

GPT-5 has a tendency to "improve" on instructions. When asked to refactor a function *without changing its interface*, it changed the return type. When asked to write a test for a specific function only, it wrote tests for adjacent functions too. Helpful in casual use; problematic when precision matters.

### Hallucinated APIs

All three models hallucinate. GPT-5 hallucinated specific library methods in 3/14 tasks, always confidently. Claude hallucinated in 2/14. This is still the most dangerous failure mode for production use.

## Benchmark vs reality

OpenAI's published benchmarks show GPT-5 leading on MMLU, HumanEval, and MATH. These translate to real-world gains on structured tasks with clear right answers.

They don't capture:
- Instruction precision
- Security reasoning
- Knowing when to say "I don't know"

## Verdict

GPT-5 is the best general-purpose model for document analysis and code generation on mainstream frameworks. For security work and tasks requiring strict instruction following, Claude remains stronger. Neither should be used without human review for production code.

The gap between models is narrowing. In six months, this comparison will look different.
